Appendix B — SDK Coverage Map

This book claims to teach the Playdate Lua SDK, and a claim like that deserves an audit. This appendix is the audit: every subsection of the API reference in Inside Playdate (§7, SDK 3.0.6) appears in the tables below, mapped to the chapter — and the chapter section, by its heading — where the book covers it, with an honest depth rating. Use it in either direction. Coming from the SDK reference (“what does the book say about playdate.pathfinder?”), scan the leftmost columns for the namespace and follow the cross-reference. Coming from a gap in your own game (“I need a scrolling list”), the area names are descriptive enough to browse.

The depth ratings mean exactly this. Full: the book teaches the area’s working core through the chapter’s buildable example, including at least one gotcha — the standard set by the authoring rules. Introduced: the API is explained and situated — what it is, when to reach for it — with real calls on the page, but the chapter’s example does not lean on it or only part of the area appears. Mentioned: named with enough context to know it exists and where it fits, nothing more. Not covered: absent from the book; every such area gets a sentence at the end of this appendix saying what it is and where to read about it, so the map has no blank edges. Ratings describe the area’s working core, not every last function — the fine print after the tables lists the stray functions inside otherwise-covered areas that the book skips.

Sixty-five areas: 39 Full, 11 Introduced, 2 Mentioned, 13 Not covered. Everything a single-player game running on the device touches — graphics, sound, input, timers, persistence, pathfinding, the lifecycle — is taught at Full or Introduced depth. Part VII moved three of those numbers: the stencil family is its own row now, because Chapter 24 builds a light compositor out of it rather than name-checking it; and JSON is promoted from Introduced to Full, because Chapter 25 treats a datastore round-trip’s key semantics as a structural constraint on the save format rather than a footnote. The Not covered column clusters at the edges: connectivity, recorded-media plumbing, and single-function conveniences. Here is what each one is, so you know whether you need it.

Localization (§7.6) is one function: playdate.getSystemLanguage() returns whether the console is set to English or Japanese, so a bilingual game can pick its strings. Accessibility (§7.7) is two reads — getReduceFlashing() and getFlipped() report the user’s system settings; a game with flash-heavy effects should check the former and route it into a master switch like the one Chapter 15 builds. Video (§7.20) plays pre-rendered .pdv files through playdate.graphics.video — for cutscenes, and rarely worth the storage in a 1-bit game. In sound, sample (§7.28) is the raw audio-buffer object beneath sampleplayer (slicing, saving, loading); signal and controlsignal are modulation sources that automate synth parameters from sequencer tracks; the effects family (bitcrusher, ring modulator, one- and two-pole filters, overdrive, delay line) attaches DSP to channels; mic input records from the built-in or headset microphone; and audio output covers headphone detection and the audio device clock. Strings (§7.29, CoreLibs/string) adds UUID() and three whitespace-trimming functions to standard Lua’s string. Math (§7.23) is exactly one function, playdate.math.lerp — this book writes a + (b - a) * t where it needs it. Networking (§7.24, OS 2.7+) brings HTTP and TCP, up to four connections, behind a user-consent prompt; serial communication (§7.33) lets a game receive msg commands over USB; and Playdate Mirror (§7.34) provides callbacks so a game can lighten its load while the Mirror app streams it to a computer. For all of these, the numbered section of Inside Playdate in the left column is the definitive reference — and after twenty-five chapters of this book’s habits, none of them will read as foreign.

Fine print on the partially covered rows: within areas rated Full or Introduced, the book does not use playdate.apiVersion() (§7.2), the calendar half of date & time (getTime, getGMTTime, epoch conversions — tabulated but not exercised in Appendix A), the display transforms setScale, setInverted, setMosaic, setFlipped, and setOffset (§7.16), or the console helpers printTable and where() (§7.14). Each behaves exactly as its reference entry says, and none hides a trap the book’s covered neighbors have not already taught you to expect.
